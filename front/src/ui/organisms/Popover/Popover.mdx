# Popover

A reusable popover component built on top of the native HTML [Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API). It is designed to be used alongside the [`usePopover`](?path=/docs/utils-hooks-usepopover--docs) hook, which links the trigger and the popover and sets up [CSS Anchor Positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning) automatically.

## Usage

```tsx
import { usePopover } from '@Front/ui/utils/hooks/usePopover/usePopover';
import { Popover } from '@Front/ui/organisms/Popover/Popover';

const Example = () => {
  const { triggerProps, popoverProps } = usePopover();

  return (
    <>
      <button type="button" {...triggerProps}>
        Open popover
      </button>

      <Popover
        {...popoverProps}
        title="Confirmation"
        primaryButtonProps={{ children: 'Confirm', onClick: () => {} }}
        secondaryButtonProps={{ children: 'Cancel' }}
      >
        Are you sure you want to proceed?
      </Popover>
    </>
  );
};
```

Spread `triggerProps` on any focusable element and `popoverProps` on `<Popover>`. The hook takes care of generating a unique ID and setting up the CSS anchor link between the two elements.

## Accessibility

- The close icon button has a built-in accessible name ("Fermer la fenêtre") and uses `popovertargetaction="hide"` to dismiss the popover via the native Popover API.
- The `popover="auto"` default enables light-dismiss (click outside or <kbd>Escape</kbd>), improving keyboard and pointer accessibility.
- The trigger element receives a `data-popover-trigger` attribute and `popoverTarget` linking it to the popover, satisfying the programmatic association requirement for assistive technologies.

## CSS Anchor Positioning

`Popover` uses [CSS Anchor Positioning](https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_anchor_positioning) to position itself relative to its trigger. The `usePopover` hook injects a `--popover-anchor-name` CSS custom property on both elements so each popover instance resolves its own anchor independently. The component includes four automatic fallback positions: right (default), left, above-right, and above-left.

> **Browser support:** CSS Anchor Positioning requires Chrome 125+. In unsupported browsers the popover will appear in its default (centered) browser position.

## Portal rendering

For the same reason as `Modal`, `Popover` should be rendered via a **React Portal** into `document.body` when stacking context or landmark pollution is a concern:

```tsx
import { createPortal } from 'react-dom';

{createPortal(
  <Popover {...popoverProps} title="Confirmation" primaryButtonProps={{ children: 'Confirm' }}>
    Content
  </Popover>,
  document.body,
)}
```
